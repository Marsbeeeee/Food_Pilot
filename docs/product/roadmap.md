## 阶段路径

1. 第一阶段：完成基础场景闭环
2. 第二阶段：增强品牌与配置精度
3. 第三阶段：强化个体化与长期使用价值
4. 第四阶段：探索平台或生态接入

## 当前优先级分层

| 优先级 | 目标 | 对应任务 |
| --- | --- | --- |
| P0 | 完成“场景输入到决策输出”最小闭环 | Task 1 ~ Task 5 |
| P1 | 提升品牌感知、配置识别和截图能力 | Task 6 ~ Task 8 |
| P2 | 提升长期使用价值和多入口扩展能力 | Task 9 ~ Task 10 |

## 建议执行顺序

1. 统一输出契约
2. 建立输入层与商品理解层
3. 建立品牌感知估算与个体化决策
4. 升级前端工作台
5. 补强 OCR、置信度和模板体系
6. 打通比较、复盘、评估与运营

## P0 完成定义

1. 用户可直接输入商品标题或描述。
2. 系统已有最小可用的“品类 -> 品牌 -> 品牌商品”分层目录。
3. 系统能输出统一决策卡片。
4. 结果包含估算、推荐等级、风险提示、个体适配说明和至少一条建议。
5. 高频品类至少覆盖奶茶、咖啡、汉堡/炸鸡、便当四类。
6. 对未收录商品有可用的私有补录与收藏路径。
7. 低置信输入进入澄清或降级，不伪装成高确定结果。
8. 前端具备真实点单场景可用的移动端界面。
