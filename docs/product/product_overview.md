## 目标

定义 FoodPilot 的长期产品方向、当前阶段定位、差异化价值与明确边界。

## 产品定位

FoodPilot 是一个面向外卖、即时零售与电商点单场景的 AI 饮食决策助手，帮助用户在下单前理解商品结构、估算营养、判断是否适合当前目标，并给出风险提示与可执行建议。

当前核心价值不是“自动记全量饮食流水”，而是：

1. 用户通过 Chat 或点单决策模式提交商品信息。
2. 系统输出结构化的估算和决策结果。
3. 用户按需把结果显式保存到 Food Log。
4. 用户基于保存结果进入 Insights 做日/周分析。
5. Profile 为推荐、安全约束和个体化判断提供上下文。

## 目标用户

1. 有减脂、控糖、增肌、维持体重等明确目标的用户。
2. 经常点外卖、喝饮品、购买即时零售商品的用户。
3. 需要在多个商品之间快速做决定，但缺少判断依据的用户。
4. 希望先获得点单层面建议，而不是从完整饮食记录开始的轻量用户。

## 典型场景

1. 点单前快速判断
2. 多商品比较
3. 品牌差异判断
4. 个体适配判断

## 当前阶段产品形态

当前阶段以“外部场景助手”为主，不依赖平台内部插件或平台级权限。用户通过外部独立页面，以商品标题、商品描述、截图等方式把平台商品带入系统，再由系统完成理解、估算与建议输出。

## 关键差异化方向

1. 品牌感知型估算
2. 个体化适配结论
3. 场景化输入支持
4. 可执行建议

## 当前边界

当前阶段明确不追求：

1. 医疗级准确性
2. 平台内插件能力作为前提
3. 一次性覆盖全平台、全品牌、全商品
4. 只有静态数值而没有决策建议的产品表达

## 对外一句话描述

FoodPilot 当前是一款以聊天与场景输入为主入口的营养决策助手，帮助用户在点单前理解商品结构、估算营养风险，并结合个人目标做出更稳妥的选择。
